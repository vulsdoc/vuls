Tutorial - Local Scan Mode

This tutorial will let you scan the vulnerabilities on the localhost with Vuls. This can be done in the following steps.

  1. Launch CentOS
  2. Deploy Vuls
  3. Configuration
  4. Check config.toml and settings on the server before scanning
  5. Scan
  6. Reporting
  7. TUI(Terminal-Based User Interface)
  8. Web UI (VulsRepo)

Step1. Launch CentOS7

  • We are using the old AMI for this example
  • Add the following to the cloud-init, to avoid auto-update at the first launch.

    • Q: How do I disable the automatic installation of critical and important security updates on initial launch?

      #cloud-config
repo_upgrade: none

Step2. Deploy Vuls

There are several ways to set up Vuls.

An easy one is vulsctl/install-host. It can be tedious, but you can also do set up manually instead of vulsctl.

Step3. Configuration

Create a config file(TOML format).

$ cd $HOME
$ cat config.toml
[servers]

[servers.localhost]
host = "localhost"
port = "local"

Step4. Check config.toml and settings on the server before scanning

$ vuls configtest

see Usage: configtest

Step5. Start Scanning

$ vuls scan
