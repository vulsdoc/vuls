VulsRepo

VulsRepo is awesome OSS Web UI for Vuls. With VulsRepo you can analyze the scan results like Excel's pivot table.

Online Demo

http://usiusi360.github.io/vulsrepo/

Requirements

  • future-architect/Vuls >= v0.4.0
  • Web Browser : Google Chrome or Firefox

インストール

Vulsのホームフォルダは「/opt/vuls」であることを前提にして記載

Step1. VulsでJSON形式のファイルを出力

<br />$ vuls scan 
$ vuls report -format-json

「/opt/vuls/results/」にJSONファイルが出力される。

Step2. VulsRepoのインストール

From now on, executed by a user running the vuls scan.

  • GitHubからクローン


    $ cd $HOME $ git clone https://github.com/usiusi360/vulsrepo.git

Step3. vulsrepo-serverの設定を変更

環境に合わせてパスを設定する

<br />$ cd $HOME/vulsrepo/server
$ cp vulsrepo-config.toml.sample vulsrepo-config.toml
$ vi vulsrepo-config.toml
[Server]
rootPath = "/home/vuls-user/vulsrepo"
resultsPath  = "/opt/vuls/results"
serverPort  = "5111"
  • ResultsPath に対してシンボリック リンクのパスを設定してはいけない。

Step4. vulsrepo-serverを起動

<br />$ pwd
$HOME/vulsrepo/server

$ ./vulsrepo-server 
2017/08/28 11:04:00 main.go:90: INFO: RootPath Load:  /root/work/vulsrepo
2017/08/28 11:04:00 main.go:97: INFO: ResultsPath Load:  /opt/vuls/results
2017/08/28 11:04:00 main.go:66: Start: Listening port: 5111
  • Linux（64 ビット）以外については自分でビルドする必要がある。 「ビルド」セクションを参照すること。

Step5. vulsrepo-serverを常駐起動

Case: SystemV (/etc/init.d)

  • スタートアップ ファイルをコピーする。環境に合わせて内容を変更する。


    $ sudo cp $HOME/vulsrepo/server/scripts/vulsrepo.init /etc/init.d/vulsrepo $ sudo chmod 755 /etc/init.d/vulsrepo $ sudo vi /etc/systemd/system/init.d/vulsrepo

  • 自動起動するよう設定する。


    $ sudo chkconfig vulsrepo on

  • vulsrepo-serverを起動する。


    $ sudo /etc/init.d/vulsrepo start

Case: systemd (systemctl)

  • スタートアップ ファイルをコピーする。環境に合わせて内容を変更する。


    $ sudo cp $HOME/vulsrepo/server/scripts/vulsrepo.service /lib/systemd/system/vulsrepo.service $ sudo vi /lib/systemd/system/vulsrepo.service

  • 自動起動するよう設定する。


    $ sudo systemctl enable vulsrepo

  • 設定が正しいかチェックする。


    $ sudo systemctl list-unit-files --type=service | grep vulsrepo vulsrepo.service enabled

  • vulsrepo-serverを起動する。


    $ sudo systemctl start vulsrepo

使い方

ブラウザでアクセスする。

<br />http://<server-address>:5111

ダイジェスト認証

  1. ダイジェスト認証を使う場合、認証ファイルを作成する。


    $ ./vulsrepo-server -h Usage of ./vulsrepo-server: -c string AuthFile Path (default "/home/vuls-user/.htdigest") -m make AuthFile -r string realm (default "vulsrepo_local") -u string login user (default "vuls")

    ex) $ ./vulsrepo-server -m Password: **** AuthFile Path : /home/vuls-user/.htdigest realm : vulsrepo_local login user : vuls 2017/08/28 19:11:59 main.go:96: Create Success

  1. vulsrepo-config.tomlを編集する。


    $ vi vulsrepo-config.toml [Auth] authFilePath = "/home/vuls-user/.htdigest" realm = "vulsrepo_local"

  1. vulsrepo-serverを起動する。

SSL を使用

  1. 自己署名証明書を作成


    $ openssl genrsa -out key.pem 2048

    $ openssl req -new -x509 -sha256 -key key.pem -out cert.pem -days 3650

  1. vulsrepo-config.tomlを編集する。


    $ vi vulsrepo-config.toml [Server]

    serverSSL = "yes" serverCert = "cert.pem" serverKey = "key.pem"

  1. Vulsrepo-serverを起動する。

Vulsrepo-serverをビルド

  • Linux（64bit）以外は自分でビルドする必要がある。

  • 事前にgolangをインストールしておくこと。


    $ mkdir -p $GOPATH/src/github.com/usiusi360/ $ cd $GOPATH/src/github.com/usiusi360/ $ git clone https://github.com/usiusi360/vulsrepo.git $ cd vulsrepo/server $ go get -u github.com/golang/dep/... $ dep ensure $ go build -o vulsrepo-server

SETTING

  • ピボットテーブル
設定名
Show / Hide the Itemピボット テーブルで表示する項目を設定する。Summary、CVSS Metricsのデータは非常に大きいため、脆弱性の数が多いとエラーが発生する。その場合はOFFにすると改善する場合がある。
Priorityピボットテーブルに表示するデータの優先度を設定する。デフォルトはNVD。

設定の共有

  • 他の人と設定を共有する方法

  1. 右上にあるLinkボタンをクリック
  2. ボタンをクリックしてクリックボードにコピーする
  3. 他の人にURLを渡す
  4. 渡されたURLにブラウザでアクセスしてファイルを選択すると、ピボットテーブルは同じ設定で表示される。

FAQ

  • 何故、VulsとVulsRepoで結果の集計値が違うのか？

Vulsはホストに含まれるCveIDの数を集計している。 しかし、VulsRepoはパッケージ毎に含まれるCveIDの数をカウントしている。 もし1 つのパッケージに 1 つの以上の CveID が関連している場合、Vuls より合計は多くなる。
