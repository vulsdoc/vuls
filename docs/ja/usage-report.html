  • 9/23 Fixedは、全部で23の脆弱性があること、そしてそのうち9個が修正済みであること、残りの14個が未修正であることを意味します。
  • 285 updatable packagesは、スキャン対象サーバに285個のアップデート可能なパッケージがあることを意味します。

  • Max Score は、Max CVSS Scoreを意味します。

  • nvd はNVDの CVSS Vector です。

  • redhat はRed Hat OVALの CVSS Vector です。

  • jvn はJVNのCVSS Vector です。

  • CWE はそのCVEのCWE - Common Weakness Enumeration を意味します。

  • [OWASP Top10] はそのCWEが OWASP TOP 10 にあることを意味します。

  • Affected PKG はその脆弱性を含むパッケージのバージョンの情報を示します。

  • Confidence はその検知の信頼度を意味します。

    • 100 は信頼度が高い。
    • YumUpdateSecurityMatch はその脆弱性を検知した手法です。

  • Confidence のリスト

    検知方法信頼度OS説明

    YumUpdateSecurityMatch100RHEL, Amazon, Oracleyum-plugin-securityによる検知
    ChangelogExactMatch95CentOS, Ubuntu, Debian, Raspbianチェンジログに記載されているバージョンとそのパッケージのバージョンの完全一致
    ChangelogLenientMatch50Ubuntu, Debian, Raspbianチェンジログに記載されているバージョンとそのパッケージのバージョンの大まかな一致
    PkgAuditMatch100FreeBSDpkg auditによる検知
    CpeNameMatch100Allconfig.tomlに記載されたCPEを用いたNVDの検索

Example: Generate all client scan reports

# Show scan history
$ vuls history

# Generate reports for all scan history
$ for REPORT_DATE in $(vuls history | awk '{ print $1 }') ; do echo "$REPORT_DATE" | vuls report -format-one-line-text -pipe ; done

# Generate reports for a specific date
vuls history | grep "DATE" | vuls report -format-one-line-text -pipe

Example: Put results in S3 bucket

To put results in S3 bucket, configure following settings in AWS before reporting.

  • S3バケットを作る。Creating a Bucketを参照。
  • S3のリソースへのアクセスを設定する。いくつかの方法で出来ます。
    • 環境変数を設定する。Configuring the AWS Command Line Interfaceを参照。
    • セキュリティクレデンシャルを設定する。Configuring the AWS Command Line Interfaceを参照。
    • サービス用のIAMロールを作成し、サービス(EC2, AWS Lambda) にアタッチ Creating a Role to Delegate Permissions to an AWS Service
  • 環境変数やセキュリティクレデンシャルを設定するには、アクセスキーを作成する必要があります。 Managing Access Keys for IAM Usersを参照。

Example of IAM policy:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:ListAllMyBuckets"
            ],
            "Resource": "arn:aws:s3:::*"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::vuls/*"
        }
    ]
}

config.toml

[aws]
profile = "default"
region = "ap-northeast-1"
s3Bucket = "vuls"
s3ServerSideEncryption = "AES256"

reporting

$ vuls report \
      -to-s3 \
      -format-json

With this sample command, it will ..

Put scan result(JSON) in S3 bucket

  • AES256を使用
  • バケットの名前は「vuls」
  • リージョンはap-northeast-1
  • プロファイルは「default」
  • サーバサイドの暗号化アルゴリズム。（例: AES256、aws:kms）

Example: Put results in Azure Blob storage

To put results in Azure Blob Storage, configure following settings in Azure before reporting.

  • Azure Blob Containerを作る

config.toml

[azure]
accountName = "default"
accountKey = "xxxxxxxxxxxxxx"
containerName "vuls"

$ vuls report -to-azure-blob
...

With this sample command, it will ..

Put scan result(JSON) in Azure Blob Storage.

  • コンテナ名は「vuls」
  • ストレージのアカウントは「test」
  • アクセスキーは「access-key-string」

account and access key can be defined in environment variables.

$ export AZURE_STORAGE_ACCOUNT=test
$ export AZURE_STORAGE_ACCESS_KEY=access-key-string
$ vuls report -to-azure-blob

ignorePkgsRegexp = ["^vim"]

Example: Add optional key-value pairs to JSON

Optional key-value can be outputted to JSON. The key-value in the default section will be overwritten by servers section's key-value. For instance, you can use this field for Azure ResourceGroup name, Azure VM Name and so on.

  • config.toml
[default]
[default.optional]
key1 = "default_value"
key3 = val3


[servers.bsd]
host     = "192.168.11.11"
user     = "kanbe"
[servers.bsd.optional]
key1 = "val1"
key2 = "val2"
  • bsd.json
[
  {

    "Family": "FreeBSD",
    "Release": "10.3-RELEASE",

    "Optional": {
        "key1": "val1" ,
        "key2": "val2" ,
        "key3": "val3"

  }
